Чистый код DAO (Data Access Object, объект доступа к данным) означает,
что логика работы с данными в приложении абстрагирована и инкапсулирована в специальных объектах DAO.

В программном обеспечении объект доступа к данным (DAO) —
это шаблон, предоставляющий абстрактный интерфейс для какого-либо типа базы данных или
другого механизма сохранения данных. Сопоставляя вызовы приложения с уровнем сохранения данных,
DAO предоставляет некоторые конкретные операции с данными, не раскрывая подробностей о базе данных.
Такая изоляция поддерживает принцип единой ответственности. Она отделяет доступ к данным,
необходимый приложению, в терминах объектов и типов данных, характерных для предметной области
(публичный интерфейс DAO), от того, как эти потребности могут быть удовлетворены с помощью
конкретной СУБД, схемы базы данных и т. д. (внедрение DAO).
